# Hackathon Playbook

Use this repo when you need to ship something impressive fast.

## Day 0 Setup

1. Edit `src/siteConfig.js`
2. Update `.env`
3. Replace `public/logo.png` and `public/favicon.ico`
4. Update content in `src/siteContent.js` if you need SEO pages
5. If you need product functionality, add it in `backend/` and call it from `src/lib/apiClient.js`

## Fastest Winning Flow

1. Pick one painful problem.
2. Write one sharp headline and one clear CTA.
3. Build one magical demo flow.
4. Add one measurable proof point.
5. Make the landing page feel premium.
6. Keep the backend thin and judge-friendly.

## Recommended Build Order

1. Landing page
2. Demo endpoint
3. Main user workflow
4. Analytics / proof / screenshots
5. SEO and deploy assets

## What Judges Usually Reward

- clarity over complexity
- visible product taste
- one obvious use case
- credible technical execution
- smooth demo story
